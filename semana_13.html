TOMCAT Y JSP

En esta semana 13 hablaremos acerca de TOMCAT y JSP

DEFINICIONES


Cliente en JSP:
En JSP, el término "cliente" se refiere al navegador web o aplicación que solicita una página web al servidor Tomcat. El cliente envía una solicitud HTTP al servidor Tomcat que contiene la URL de la página JSP solicitada. .

Apache Tomcat: Es un servidor web y contenedor servlet de código abierto desarrollado por la Apache Software Foundation. Tomcat actúa como un servidor web que sirve páginas web estáticas y dinámicas.
Metodos HTTP:GET, POST, PUT y DELETE se utilizan para determinar la forma en que se procesa una solicitud HTTP enviada desde el cliente al servidor.

  • GET: Este método se utiliza para solicitar datos del servidor. En JSP, cuando se recibe una solicitud GET, los parámetros de la solicitud pueden ser accesibles a través del objeto
  • POST: Este método se utiliza para enviar datos al servidor, generalmente a través de un formulario HTML en una página JSP. En este caso, los datos enviados por el cliente pueden ser accesibles a través del objeto
  • PUT y DELETE: Estos métodos HTTP son menos comunes en formularios HTML estándar, pero pueden ser utilizados en aplicaciones web más avanzadas o cuando se trabaja con APIs RESTful.

PROCEDIMIENTO


Para esta semana el procedimiento que se tuvo en cuenta fue aplicar conocimientos en JSP como:

  • Realizar conexión a base de datos
  • Usar metodo GET, POST, PUT, DELETE


RESULTADOS


Con la practica y la guia del ingeniero pudimos obtener los siguientes resultados:

  • Realizar pagina web haciendo uso de JSP y HTML
  • Usar metodos GET y POST , ademas de diferenciarlos
  • Hacer uso de formularios donde se envien el metodo de GET y POST
